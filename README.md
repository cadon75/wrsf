# WRSF

主にMMLなどから作製されたソフトウェア音源（FM音源等）サウンドのwavファイルをターゲットとした  
懐かしい感じの音に変換する変換型簡易誰得フィルタです。

## 動作環境 ( 動作確認済環境 )
*   **OS**: Windows 11 (x86_64) / Linux (Ubuntu系 24.04)
*   **依存関係**: Linuxでは使用する環境によっては libgtk-3-dev 等が必要になる事があります。  
ターミナルで操作しますが、GUI(ExplorerやLinuxでは各種ファイラーなど)の機能も使用します。

## 主な使い方 / 機能
*   主に  
「昔の機器からライン入力されたラジカセで録音されたカセットテープから再生してるっぽい音」  
にするのがメインです。(ぽいが重要)
*   変換できるファイルは  
**wavファイル (22.5/44.1/48/96kHz ・ int 16/24/32bit / float 32bit ・ 2ch-Stereo) のみ** です。  
フィルタは 44.1kHz - 16bit int - Stereo をメインターゲットで計算しています。  
他の対応はオマケですが、入力される音次第ではメインターゲットより良くなるかも（いい加減）    
*   GUI上のターミナルからバイナリを起動すれば、対話方式で処理方法を選択できます。
*   バイナリにファイル名を付けて起動（コマンドライン起動）すれば基本フィルタ処理をすぐに一気に行います。
*   書き出すwavファイルは、上書き確認あり（コマンドライン起動時は強制上書き）の追記自動命名です。  
都度ユーザーが対応して下さい。

*   **フィルタ**  
    1. 再生特性フィルタ  
    レトロな機器の音の変化特性をwavファイルに簡易的に付与します。  
    2. カセットテープフィルタ  
    カセットテープの特性をwavファイルに簡易的に付与します。  
    3. アナログ特性フィルタ  
    アナログ機器が持つ音の変化の特性をwavファイルに付与します。  
    4. スピーカーフィルタ  
    ラジカセのスピーカーの様な音の変化の特性をwavファイルに付与します。  
    5. レコードノイズ  
    レコードノイズっぽいものを付与します。

* 最初の選択で1番（コマンドライン -1） の基本フィルタは上記の1～4のフィルタを  
2番（コマンドライン -2）は上記の1～5全部のフィルタ処理を行います。  
コマンドライン起動時のファイル名とコマンド引数は前後どっちでもOKです。  

* 基本的に1番の基本フィルタのデフォルト設定で大丈夫ですが、  
元のwav次第では高音域が強過ぎと思う場合は、対話式起動で再生特性フィルタのみ2番に変えてみて下さい。  
再生特性フィルタ3番は更に高域を求める時、4番5番は特殊系です。  

*   簡易フィルタソフトですが、やってる事はマスタ同等で中身は結構重い処理もしています。  
現代的な環境なら数秒ですが、読み込むwavファイルや環境によっては処理に時間がかかります。  
（LinuxビルドではWindowsビルドよりも処理に時間がかかる傾向があります）  

## ビルド
*   リポジトリをクローンしてから、Windows / Linux 上 の RUST (作製時は RUST 1.95.0) で  
それぞれ使用環境に合わせてビルドするだけです。   

```
    Windows : cargo build --release --target x86_64-pc-windows-msvc
    Linux   : cargo build --release --target x86_64-unknown-linux-gnu
```    
## 注意事項
* あくまで作者の個人的補助用に作製したソフトです。  
精度や忠実なリアル感は求めないで下さい。  

* 一応、使うフィルタの設定などを出来るよう（3番の詳細設定）にしていますが  
出来るようにしてあるだけのオマケの未検証で組み合わせによっては壊れた音になります。  

* 副産物的な仕様で、完全無音のwavを読み込ませると  
1番の基本フィルタでCRT砂嵐っぽい音、2番の全フィルタでレコードノイズのみが生成されますが  
あくまでフィルタの副産物であって狙った機能ではないのでご注意下さい。

* 動作中は結果を書き出すまでオンメモリ動作です。  
例で、96kHz24bit-5分のwavファイルで計算上で2.7G程メモリを使います（殆ど変換バッファメモリ）  
普通の使い方なら現代的な環境なら問題ないはずですが  
あまりにも巨大サイズのwavファイルやメモリが極端に少ない環境ではご注意下さい。  

* 個人ソフトなのでSmartScreenなどが反応する可能性が高いです。  
免責の通りなので信用出来ない場合はご利用を控えるか、各環境でビルドしてください。
